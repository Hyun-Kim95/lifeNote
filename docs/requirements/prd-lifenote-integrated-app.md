# PRD — 통합 생활 관리 앱 (lifeNote)

- 문서 상태: **승인됨** (2026-04-14, 채팅 컨펌)
- 최종 수정: 2026-04-14
- 확정 사항: **웹과 앱을 동시에 개발**한다. 비즈니스 규칙·API는 공통을 우선하고, 표현·탐색은 플랫폼별로 달라도 된다(프로젝트 규칙 `20-web-vs-app` 준수).

## 1. 제품 목표

- **목표**: 계획·할 일·일기·가계·공지·커뮤니티를 한곳에서 관리하고, 관리자가 콘텐츠·회원·대표 시간표를 운영할 수 있는 서비스를 제공한다.
- **성공 지표(초안)**: DAU/재방문, 일기·계획 작성 완료율, 커뮤니티 참여(게시·댓글), 관리자 운영 효율(공지·배너 반영 시간) — 구체 수치는 출시 전 합의.

## 2. 사용자 유형

| 유형 | 설명 |
|------|------|
| 일반 회원 | 계획표, To-do, 일기, 가계부, 명언 배너, 공지 열람, 커뮤니티 참여, SNS 로그인, 통계 열람 |
| 관리자 | 공지·배너·팝업·회원·커뮤니티·일기 템플릿·메뉴·관리자 계정, 대표 시간표(세그먼트별) 운영 |
| (선택) 운영/슈퍼관리자 | 관리자 권한 위임, 감사 로그 — 범위 미확정 |

## 3. 핵심 기능 범위 (요청 반영)

### 3.1 계획표

- 개인 주간/월간 계획 생성·편집·삭제.
- **커뮤니티 연계**: 계획표 공유(전체 또는 일부 블록), 댓글·반응 정책은 미확정.

### 3.2 To-do 리스트

- 목록·우선순위·기한·완료 처리.
- **완료율 표시**: 목록/대시보드에서 완료 비율(예: N/M, %) 표시.

### 3.3 일기

- 날짜별 작성, (관리자 제공) **일기 템플릿** 선택 적용.
- 비공개 기본, 공개 범위가 필요하면 정책 합의.

### 3.4 명언 배너

- 앱 내 배너 영역에 명언(문구·출처) 노출.
- **관리자 배너관리**와 연동(노출 순서, 기간, ON/OFF).

### 3.5 가계부 (식비 중심 예시)

- **월 식비 예산** 입력.
- **일별 식비 사용** 입력.
- **남은 사용 가능 금액** = 월 예산 − (해당 월 누적 사용) — 일 단위로 갱신 표시.
- 사용자 언급 **메모장 사용처**: 일별 지출에 메모(용도·장소 등) 필드 제공 여부 — **미확정(권장: 일별 항목에 메모 필드)**.

### 3.6 공지사항

- 목록·상세, 중요 공지 상단 고정(관리자 옵션).
- 푸시/알림 연동 여부 — **미확정**.

### 3.7 커뮤니티

- 계획표·To-do **공유** 및 댓글 등 소통.
- 신고·차단·비공개 게시물 정책 — **미확정(필수 권장)**.

### 3.8 SNS 로그인

- 소셜 계정으로 가입·로그인, 기존 계정 연동 정책.
- **1차 SNS: 구글 로그인으로 확정**(2026-04-14). 이후 카카오·애플 등은 동일 OAuth 교환 패턴으로 확장.

### 3.9 통계

- To-do 완료율, 일기 작성 일수, 가계부 지출 추이, 계획 달성률 등 집계.
- 기간 필터(주·월·년) — **세부 지표는 1차 이후 확장 가능**.

### 3.10 관리자 — 대표 시간표

- **나이대, 취업 여부** 등 속성(세그먼트)별 **대표 시간표 템플릿** 생성·배포.
- 일반 사용자가 “내 프로필에 맞는 대표표를 불러오기” 흐름 — **미확정(권장)**.

## 4. 관리자 메뉴 (요청 목록과 매핑)

| 메뉴 | 주요 기능 |
|------|-----------|
| 회원관리 | 조회, 상태(활성/정지), 역할, SNS 연동 여부, (선택) 강제 로그아웃 |
| 관리자관리 | 관리자 계정·역할·권한(메뉴별) |
| 메뉴관리 | 앱/웹 네비게이션 항목, 노출 순서, 권한별 가시성 |
| 일기템플릿관리 | CRUD, 카테고리, 노출 순서 |
| 배너관리 | 명언 배너 CRUD, 기간·우선순위 |
| 공지사항관리 | CRUD, 고정, 게시 기간 |
| 커뮤니티관리 | 게시글·댓글 조회/숨김/삭제, 신고 큐(도입 시) |
| 팝업관리 | 기간·대상(전체/세그먼트)·빈도, ON/OFF |

**대표시간표**는 별도 상위 메뉴 또는 “콘텐츠 관리” 하위로 둘지 **미확정**.

## 5. 주요 사용자 흐름 (요약)

1. **온보딩**: SNS 로그인 → (선택) 프로필(나이대·취업여부) → 대표 시간표 추천.
2. **일일 사용**: 홈(명언 배너·오늘 To-do·남은 식비 요약) → 계획/일기/가계부 입력.
3. **커뮤니티**: 공유 글 작성 → 댓글 알림(선택).
4. **관리자**: 공지/배너/팝업 반영 → 회원 이슈 처리 → 커뮤니티 모더레이션.

## 6. 화면·상태 UI (Gate 1 스펙 수준)

각 주요 화면은 다음 상태를 정의한다: 기본, 로딩, 빈 데이터, 오류, 권한 제한.

- 홈, 계획표 편집, To-do, 일기(작성/목록), 가계부(월·일), 공지 목록/상세, 커뮤니티 피드/상세, 통계, 설정·프로필.
- 관리자: 위 메뉴별 목록·폼·미리보기(배너/팝업).

## 7. 정책·예외 (초안)

- **개인정보**: 일기·가계부는 타인에게 기본 비공개. 커뮤니티 공유는 사용자 명시 동작 시에만.
- **식비 가계부**: 월 경계 시 이월/초기화 규칙 — **미확정(권장: 월별 독립 예산)**.
- **관리자 권한**: 최소 권한 원칙, 역할별 메뉴 가시성.

## 8. API 계약 (Gate 2)

**단일 기준 문서**: `docs/requirements/api-contract-lifenote-v1.md` (v1, 2026-04-14)

요약:

- 접두사 **`/v1`**, 인증 **Bearer JWT**, SNS는 **`POST /v1/auth/oauth/exchange`** 등으로 교환.
- 오류 본문 공통: `{ "error": { "code", "message", "details?", "requestId?" } }` + HTTP 상태 표준 매핑.
- MVP: To-do, 주간 계획, 일기, 식비 월·일, 공지 읽기, 명언 배너, 홈 요약(선택), 통계 요약, 관리자(공지·배너·회원·일기 템플릿) + 페이지네이션(관리자 **pageSize 기본 15**).
- 커뮤니티·팝업·대표시간표 등은 문서 내 **2·3차 예약** 절 — 범위 확정 시 동일 규칙으로 확장.

상세 필드·예시 JSON은 전부 위 계약서를 따른다.

### 9.1 웹·앱 역할 분리 (요약)

- **웹**: 정보 탐색·관리자 화면(테이블·필터·페이지네이션) 효율을 우선.
- **앱**: 스크롤·리스트·하단 탭 등 모바일 연속 탐색을 우선. 동일 화면 구조를 억지로 맞추지 않음.

## 10. 구현 단계 제안 (MVP → 확장)

**MVP (1차)**

- SNS 로그인 1종(**구글**) + 이메일(선택)
- To-do + 완료율, 간단 계획표(주간)
- 일기(템플릿 없이도 가능하면 템플릿은 관리자만 선행)
- 가계부 식비(월 예산·일 지출·잔액·메모)
- 공지 열람, 명언 배너(관리자 입력), 관리자: 공지·배너·회원 기본

**2차**

- 커뮤니티(공유·댓글), 통계 강화, 팝업, 대표 시간표 세그먼트

**3차**

- 일기 템플릿 고도화, 메뉴관리·관리자관리 세분화, 신고/모더레이션, 추가 SNS

## 11. 미확정·확인 필요 목록 (구현 착수 전 우선순위)

1. ~~타겟 플랫폼(웹/앱)~~ → **웹+앱 동시로 확정.** 1차 출시 범위(MVP에 커뮤니티 포함 여부)만 남음.
2. ~~SNS 1종~~ → **구글**. 계정 통합(동일 이메일 병합 등)은 여전히 합의 필요.
3. 커뮤니티 모더레이션·신고 최소 규칙.
4. 대표 시간표의 “세그먼트” 정의(필수 속성 목록).
5. 가계부: 다른 카테고리 확장 여부, 월 이월 규칙.
6. 푸시 알림 필요 여부.
